• A custom tag app-product is declared using the <:define> directive.
  • Where it's used, it's replaced with its definition and initialized with the provided values.
  • In the definition we put what's common to all products — there will be a single place to edit for changing the visualization of all products.
  • In usages we only put what's specific to each product — the code is short and uncluttered.
Custom tags are HTML macros
  • Only their usages are included in the output pages, not their definitions.
  • They're similar to Web Components, but they support legacy browsers and are much simpler to use.
  • Their name must include a -.
  • They're replaced with div tags unless a different base tag is specified.
